Aula 14 - Tipos Abstratos de Dados (TADs)

Motivação e objetivos

  • Motivação
    • alocação sequencial possuem algoritmos mais simples, porém o recurso memória pode constituir uma desvantagem pois nem sempre a memória armazenada estará em uso
    • Alocação encadeada ou dinâmica, por outro lado, minimiza o problema de desperdício de memória uma vez que blocos de memória são alocados (ou desalocados) quando necessários (ou dispensados)
  • Objetivos
    • Apresentar como listas podem ser implementadas usando alocação dinâmica de memória

TAD Lista

  • Um TAD é definido por dados e operações (que modificam os dados)
  • Especificando a Lista de forma abstrata 
    TAD Lista <tipo>
    - dados:
        quantidade: inteiro 
        valores:    Sequência de elementos do tipo <tipo>
    - operações:
        acesso(pos)        -> <tipo>
        busca(elm)         -> inteiro
        inserção(elm)      -> booleano
        inserção(pos, elm) -> booleano
        inserção(ref, elm) -> booleano
        remoção(pos)       -> booleano
        remoção(ref)       -> booleano
        junção(lst1, lst2) -> booleano
        ...

Formas de implementar listas

  • Um TAD pode ser implementado de diferentes formas
  • A lista pode ser implementada de forma
    • Sequencial
    • Encadeada (alocação dinâmica de memória)

Operações e complexidade

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Acesso (pos) \(O(1)\) \(O(pos)\) \(O(1)\) \(O(pos)\)
Busca (elm) \(O(n)\) \(O(n)\) \(O(log_2 n)\) \(O(n)\)
Inserção (elm) \(O(1)\) \(O(1)\) \(O(n)\) \(O(n)\)
Inserção (pos, elm) \(O(n-pos)^*\) \(O(pos)\) - -
Inserção (ref, elm) \(O(n - pos(ref))^*\) \(O(1)\) - -
Remoção (pos) \(O(n-pos)^*\) \(O(pos)\) \(O(n-pos)\) \(O(pos)\)
Remoção (ref) \(O(n-pos(ref))^*\) \(O(1)\) \(O(n-pos(ref))\) \(O(1)\)
Junção (lst1, lst2) \(O(min(n,m))\) \(O(1)\) \(O(n+m)\) \(O(n+m)\)

Acesso

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Acesso (pos) \(O(1)\) \(O(pos)\) \(O(1)\) \(O(pos)\)

  • Em listas sequenciais, o acesso ao elemento de posição pos é direto
  • Em listas encadeadas, deve-se percorrer pos elementos para acessar o elemento do posição pos

Busca

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Busca (elm) \(O(n)\) \(O(n)\) \(O(log_2 n)\) \(O(n)\)

  • Em listas não-ordenadas não há relação de ordem: tem que percorrer todos elementos
  • Em listas ordenadas, há relação de ordem e pode-se fazer uma busca binária
  • Em listas encadeadas, não há índices de acesso direto: não há como fazer busca binária

Inserção

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Inserção (elm) \(O(1)\) \(O(1)\) \(O(n)\) \(O(n)\)

  • Em não-ordenadas, pode-se inserir em qualquer lugar. Então, pode-se inserir no fim (sequencial) ou no início (encadeada)
  • Em ordenadas, a relação de ordem deve ser mantida, sendo, portanto, achar onde inserir (busca)
  • Mesmo usando busca binária na sequencial, é necessário deslocar os elementos

Inserção numa posição

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Inserção (pos, elm) \(O(n-pos)^*\) \(O(pos)\) - -

  • É válido apenas para listas não ordenadas (não há relacão de ordem)
  • Na sequencial, precisa-se deslocar os elementos de pos ao final
  • A complexidade pode baixar para \(O(1)\) se a sequência dos elementos não precisar ser mantida. Se não, pode-se enviar o elemento em pos para o fim da lista
  • Na encadeada, precisa-se percorrer pos nós para saber onde inserir

Inserção numa referência

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Inserção (ref, elm) \(O(n - pos(ref))^*\) \(O(1)\) - -

  • A inserção de um elemento em uma referência de posição (endereço) também é válida apenas para listas não ordenadas
  • Na sequencial, é similar à inserção com a posição (pode ser também \(O(1)\)
  • Na encadeada, basta apontar para um novo bloco de memória alocado e fazê-lo encadear com o seguinte

Remoção dada uma posição

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Remoção (pos) \(O(n-pos)^*\) \(O(pos)\) \(O(n-pos)\) \(O(pos)\)

  • Na sequencial (ordenada ou não), os elementos depois de pos devem ser deslocados
  • Obs: Novamente, pode-se fazer O(1), passando o último para pos
  • Na encadeada, é necessário percorrer pos para saber quem deve ser removido

Remoção dada uma referência

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Remoção (ref) \(O(n-pos(ref))^*\) \(O(1)\) \(O(n-pos(ref))\) \(O(1)\)

  • Raciocínio similar ao anterior...

Junção de duas listas

Listas Não-ordenadas Ordenadas
Sequencial Encadeada Sequencial Encadeada
Junção (lst1, lst2) \(O(min(n,m))\) \(O(1)\) \(O(n+m)\) \(O(n+m)\)

  • Considerando que não haverá overflow, a junção na sequencial não-ordenada é copiar os elementos da menor sequência no final da maior
  • Na encadeada não-ordenada, basta fazer com que o último de um aponte para o início do outro
  • Nas ordenadas, em ambos os casos, será necessário percorrer os elementos das duas listas para manter a relação de ordem

Qual é a melhor forma de implementar?

  • Depende... vantagens e desvantagens devem ser analisadas em função da aplicação
  • Além das complexidades apresentadas, deve-se verificar o uso de memória
  • De uma forma geral, listas encadeadas são mais convenientes quando se requer flexibilidade (no uso de memória)

Listas simplesmente encadeadas

  • Cada elemento (dado) é armazenado em um componente maior chamado nó
  • Além do dado, o nó possui uma referência para o nó seguinte
  • É necessário termos a referência para o 1o elemento da lista

  • typedef struct node {
    int          data;
    struct node* next;
} Node;

Node* list;

op1: acesso a um elemento

  • Recebe o 1o nó da lista e a posição na qual o elemento que deve ser retornado se encontra
  • Percorre a a lista a partir do 1o nó (list), acessando-se sucessivamente os campos next de cada nó da lista
  • O laço para quando pos chegar a zero (já fez pos iterações) ou chegar ao fim da lista (list é NULL)
    • Node* getElm(Node* list, int pos) {
    if (pos <= 0)
        return NULL;

    while(--pos && list) {
        list = list->next;
    }
    return list;
}

op2: busca por um elemento

  • Percorre os elementos da lista, atualizando o contador i e acessandoe sucessivamente os campos next
  • Quando o dado for o mesmo que o da lista, então achou na posição i
  • Se não achou nenhum, é porque não encontra-se no doc.
    • int getPos(Node* list, int data) {
    for (int i = 1; list; i++, list = list->next)
        if (list->data == data)
            return i;

    return -1;
}

op3: inserção no início

  • Aloca espaço dinamicamente para mais um nó e faz o next dele apontar para o antigo início da lista
  • Novo início deve ser agora o novo elemento criado
    • int add(Node* list, int data) {
    Node* aux = (Node*) malloc(sizeof(Node));
    if (aux) {
        aux->data = data;
        aux->next = list;
        list = aux;         /* isso está certo??? */
        return 1;
    }
    return 0;
}

Dica: cabeça de lista

  • Inserção de um nó especial chamado nó-cabeça
    • É o nó apontado pelo ponteiro de início da lista
    • Nunca é removido
    • Não contém valor válido
    • Ocasionalmente pode conter informações sobre a lista

op3: inserção no início (novo)

  • Usa da cabeça da lista
  • Novo início deve ser agora o novo elemento criado
    • int add(Node* list, int data) {
    Node* aux = (Node*) malloc(sizeof(Node));
    if (aux) {
        aux->data  = data;
        aux->next  = list->next;
        list->next = aux;
        return 1;
    }
    return 0;
}

op4: Remoção de uma posição

  • Usa da cabeça da lista
    • int delElm(Node *node, int pos) {
    Node *aux;

    if (pos <= 0 || ! node->next)
        return 0;  /* false: nada a remover */

    for (int i = 1; i < pos && node->next ; i++, node = node->next);

    aux = node->next;
    if (aux) {
        node->next = aux->next;
        free(aux);
    }
    return 1;
}
