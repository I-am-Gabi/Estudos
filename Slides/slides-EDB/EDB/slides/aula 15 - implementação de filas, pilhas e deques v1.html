Aula 15 - Implementação de TADs

Motivação e objetivos

  • Motivação
    • De uma maneira geral o armazenamento sequencial de listas é adequado quando a estrutura de dados requer poucas remoções e inserções
    • De preferência em situações em que estas operações não exijam movimentação de nós
    • Este é o caso de pilhas, filas e deques, onde as operações citadas são executadas nas extremidades
  • Objetivos
    • Apresentar como Pilhas e Filas podem ser implementadas usando arranjos e alocação dinâmica de memória

Listas com Restrições: Pilhas

  • Propriedade:
    • O último item inserido é o primeiro item que pode ser retirado da lista.
    • São chamadas listas LIFO ("last-in, first-out").
  • Existe uma ordem linear para pilhas, do "mais recente para o menos recente".
  • É ideal para processamento de estruturas aninhadas de profundidade imprevisível.
  • Uma pilha contém uma sequência de obrigações adiadas.
    • A ordem de remoção garante que as estruturas mais internas serão processadas antes das mais externas.

Listas com Restrições: Pilhas

  • São exemplos de uso de pilha:
    • Funções recursivas em compiladores;
    • Mecanismo de desfazer/refazer dos editores de texto;
    • Navegação entre páginas Web;
    • etc.
  • A implementação de pilhas pode ser realizada de duas maneiras:
    • através de arranjo (alocação do espaço de memória para os elementos é contígua), ou
    • através de listas encadeadas (os elementos não estão necessariamente armazenados sequencialmente na memória).

Listas com Restrições: Pilhas

  • Operações com Pilha
    • Criar pilha (create);
    • Empilhar (push);
    • Desempilhar (pop);
    • Mostrar o topo (top).

Implementação : operações

  • O conceito do que cada operação realiza é independente de implementação
  • Assim, as chamadas às funções podem ser idênticas, seja para pilhas implementadas com arranjos ou com listas encadeadas
  • Adotaremos as seguintes assinaturas: 
    /* operações sobre uma pilha (stack) */

Stack* createStack();
int push (Stack *stack, int  data);
int pop  (Stack *stack, int *data);
int top  (Stack *stack, int *data);

Estrutura usando arranjo

  • Definição da Pilha (Stack):
#define STACK_MAXSIZE 20
	   
typedef struct {
	int data[STACK_MAXSIZE];   /* dados */
	int size;                  /* serve também como ref p/ topo  */
} Stack;

Implementação usando arranjo

  • Função push( ):
/* insere um elemento no topo da pilha */
int push (Stack *stack, int data) {
	if (stack -> size < STACK_MAXSIZE) {
	    stack -> data [ stack -> size ] = data;
	    stack -> size++;
	    return 1;
	}
	return 0;
}

Implementação usando arranjo

  • Função pop( ):
/* remove e retorna o elemento do topo da pilha */
int pop (Stack *stack, int *data) {
	if (stack -> size > 0) {
	    stack -> size--;
	    *data = stack -> data [ stack -> size ];
	    return 1;
	}
	return 0;
}

Implementação usando arranjo

  • Função top( ):
/* retorna o elemento do topo da pilha */
int top (Stack *stack, int *data) {
	if (stack -> size > 0) {
	    *data = stack -> data [ stack -> size - 1 ];
	    return 1;
	}
	return 0;
}

Implementação usando lista encadeada

  • Definição da Pilha (Stack):
typedef struct node {
    int          data;
    struct node *next;
} Node;

typedef struct {
	Node *head; /* nó-cabeça */
	int   size; /* opcional */
} Stack;

Implementação usando lista encadeada

  • Função push( ):
/* insere um elemento no topo da pilha */
int push (Stack *stack, int data) {
	Node *new_head = (Node*) malloc(sizeof(Node));
	if (new_head) {
	    new_head -> data = data;
	    new_head -> next = stack -> head;
	    stack -> head    = new_head;
	    stack -> size++;
	    return 1;
	}
	return 0;	
}

Implementação usando lista encadeada

  • Função pop( ):
/* remove e retorna o elemento do topo da pilha */
int pop (Stack *stack, int *data) {
	if (stack -> head) {
	    Node *old_head = stack -> head;
	    stack -> head  = old_head -> next;
	    *data = old_head -> data;
	    free(old_head);
	    return 1;
	}
	return 0;
}

Implementação usando lista encadeada

  • Função top( ):
/* retorna o elemento do topo da pilha */
int top (Stack *stack, int *data) {
	if (stack->head) {
	    *data = stack -> head -> data;
	    return 1;
	}
	return 0;
}

Listas com Restrições: Filas

  • Propriedade:
    • É uma lista linear em que todas as inserções são realizadas em um extremo da lista, e todas as retiradas e, geralmente, os acessos são realizados no outro extremo da lista.
    • São chamadas listas FIFO ("first-in, first-out").
  • São utilizadas quando desejamos processar itens de acordo com a ordem "primeiro-que-chega, primeiro-atendido".
  • Sistemas operacionais utilizam filas para regular a ordem na qual tarefas devem receber processamento e recursos devem ser alocados a processos.

Listas com Restrições: Filas

  • São exemplos de uso de fila:
    • Controle de documentos para impressão;
    • Troca de mensagem entre computadores numa rede;
    • etc.

  • A implementação de filas pode ser realizada de duas maneiras:
    • através de arranjo (alocação do espaço de memória para os elementos é contígua), ou
    • através de listas encadeadas (os elementos não estão necessariamente armazenados sequencialmente na memória) .

Atividades

  1. Nas filas usando arranjos, a inserção de um dado pode resultar em overflow mesmo tendo espaço livre no início do arranjo. Para contornar esse problema, é possível definir filas circulares. A estratégia delas é considerar o primeiro elemento do arranjo como a continuação (próximo) do último elemento. Assim, só haverá overflow se realmente não houver mais espaço livre. Modifique a implementação da fila realizada em I.a para que ela seja circular.

Atividades

  1. O problema de TADs usando arranjos é que normalmente não se sabe o números de elementos a ser usado (para alocação prévia), gerando gastos de memória desnecessários e (talvez) problemas de overflow. Para contornar esse problema, implemente um novo TAD, chamado Vector, com as operações similares ao de um arranjo (acesso direto a um elemento a partir de um índice). Esse novo tipo deve alocar por default 10 espaços para seus dados, mas dobrar de tamanho sempre que uma operação de inserção for gerar um overflow.
