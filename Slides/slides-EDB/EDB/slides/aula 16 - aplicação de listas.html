Algoritmos de busca

Por onde começar?

Exemplo de problema
  • Controle de acesso
    Você quer entrar no DIMAp, mas a porta só abre para quem estar cadastrad no sistema. Como localizar se uma pessoa está cadastrada?
  • Definição do problema
    Seja \(A\) o conjunto de \(n\) pessoas cadastradas no sistema, pretende-se saber se a pessoa \(x \in A\)
  • Para efeitos práticos, vamos supor que \(A = \{a_0, a_1,...,a_{n-1}\}\) seja implementado como um arranjo (\(a[0], a[1]... a[n-1]\)) de \(n\) números inteiros e que \(x\) a ser procurado é também um valor inteiro.
Possíveis soluções
  • Busca aleatória
    Se garantir que um elemento não vai ser consultado mais de uma vez, a solução é dada em tempo finito. Caso contrário, pode-se não chegar a uma solução.
  • Busca sequencial
    Possui um mecanismo simples para garantir uma resposta em tempo finito.
  • Outras formas de busca???
Busca sequencial
  • Um dos métodos de busca mais simples
  • Consiste em analisar cada elemento desde o primeiro até:
    • Localizar o desejado ou
    • Analisar todos os elementos do vetor e não ter achado algum.
    • Obs: caso o arranjo esteja em ordem crescente, podemos também parar a busca quando chegar a um valor maior que o procurado.
Algoritmo e implementação
  1. Percorra o arranjo \(A\) da 1a posição até a última.
  2. Para cada posição \(i\) de \(A\), compare o valor do i-ésimo elemento de \(A\) com o valor procurado \(x\).
  3. Se, em algum momento, os valores são iguais, então \(x \in A\), caso contrário (ou seja, em nenhum momento) então \(x \notin A\).
int buscaSeq(int v[], int N, int x) {
	int i;
	for (i=0; i < N; i++) {
		if (x == v[i])
			return i;  /* posição encontrada */
	}
	return -1; /* representando que não encontrou */
};

Possível alternativa
  • Se os dados estiverem ordenados, podemos aproveitar essa informação para melhorar a busca.
    • Ordenação supõe que se pode comparar elementos em função de algum critério ou atributo do elemento (\(a < b\)).
    • Em um conjunto ordenado \(A\), para quaisquer elementos \(a_i\) e \(a_j\), se \(a_i < a_j \Rightarrow a_i < a_k\) para qualquer \(k > j\)
Possível alternativa
  • Podemos reduzir o número de elementos analisados descartando o máximo de elementos possíveis
  • Se o elemento a ser comparado for próximo a uma das extremidades do conjunto, é possível descartar poucos elementos
  • A melhor garantia é escolher o elemento do meio
Busca binária
  • Seja \(A\) um arranjo ordenado com \(n\) elementos, \(A = \{ a_0,..., a_{n-1}\}\), tal que \(a_i < a_j\) se \(i < j\), para \(i, j \in [0..n-1]\)
  • Seja \(x\) um valor a ser procurado em \(A\)
  • Algoritmo:
    1. Se \(n = 0\) então \(x \notin A\) (pois \(A\) é um arranjo vazio).
    2. Se \(x = a_{\left \lfloor n/2 \right \rfloor}\) então \(x \in A\) na posição \(\left \lfloor n/2 \right \rfloor\).
    3. Se \(x < a_{\left \lfloor n/2 \right \rfloor}\) então faça o mesmo para \(\{a_0,..., a_{\left \lfloor n/2 \right \rfloor - 1}\}\)
    4. Se \(x > a_{\left \lfloor n/2 \right \rfloor}\) então faça o mesmo para \(\{a_{\left \lfloor n/2 \right \rfloor + 1},..., a_{n-1}\}\)
Implementação (recursiva)
int buscaBin(int v[], int N, int x) {
	int i = N/2;          /* índice do elemento do meio */
	int y = v[i];         /* valor do elemento do meio */

	if (N == 0)           /* se o arranjo está vazio */
		return -1;        /* ... é porque não achou */
	else if (x == y)
		return i;                       /* achou na posição i */
	else if (x < y)
		return buscaBin(v, i, x);     /* pode estar na 1a metade */
	else                              /* x > y */
		return buscaBin(&v[i+1], N-(i+1), x); /* ou na 2a metade */
}

  • É necessário, portanto, analisar os algoritmos em termos de:

  • A esses dois itens, utilizamos os conceitos de:
