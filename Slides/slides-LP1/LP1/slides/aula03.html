Aula 3 - Depuração de Código

Objetivos da aula

  • Introduzir conceitos de teste e depuração de código
  • Para isso, estudaremos:
    • Testes de software
    • Depuração
  • Ao final da aula espera-se que o aluno seja capaz de:
    • Fazer uso de técnicas para detecção, diagnóstico e correção de erros em programas

Contexto

  • Quando programamos, geralmente inserimos erros (bugs) de diversas formas nos programas
    • Desde os mais básicos (sintáticos ou de estrutura) que levam o próprio compilador a gerar uma mensagem de erro
    • Aos erros semânticos inseridos na lógica do programa, para os quais uma simples leitura do código pode não ser suficiente

  • Neste último caso, dois procedimentos importantes são geralmente praticados para tentar encontrar uma solução
    • Testes: para descobrir os erros
    • Depuração: para localizar e eliminar os erros

Teste de software

  • Teste é um procedimento de detecção de erros em um programa
    • Bastante popular no que se refere à melhoria da qualidade
    • Testes podem ser realizados por
      • Desenvolvedores do software (testes de caixa branca)
      • Especialistas no domínio do software (testes de caixa preta)
    • Eles são classificados em diversas categorias
      • Teste unitário
      • Teste de componente
      • Teste de integração
      • Teste de regressão
      • Teste de sistema

Exemplo de teste (1)

  • Execução de uma programa que lê dois números inteiros e imprime o maior deles

  • Podemos detectar um erro no programa!!!

Exemplo de teste (2)

  • Programa que lê dois números inteiros e imprime o maior deles
#include <iostream>

int main()
{
    int num1, num2;
    std::cout << "Calculo do maior entre dois numeros" << std::endl << std::endl;
    std::cout << "Digite o primeiro numero: ";
    std::cin >> num1;
    std::cout << "Digite o segundo numero: ";
    std::cin >> num2;
    if( num1 < num2 )
        std::cout << "O maior numero eh " << num1 << std::endl;
    else
        std::cout << "O maior numero eh " << num2 << std::endl;
    return 0;
}

  • A impressão do maior numero é efetuada incorretamente!!!

    Ou o sinal de comparação está invertido!!!

Caso de teste

  • Um caso de teste é um conjunto de valores constituídos por pares de entrada e saída esperada
    • < entradas; saídas esperadas >
  • Exemplo de caso de teste para o programa que calcula o maior de dois números inteiros
    • < 100, 20; 100 >
    • < 49, 87; 87 >
    • < -85, 37; 37 >
    • < 0, -10; 0 >
    • < -15, a; entrada invalida >
    • < "lp1", 'a'; entrada invalida >

Dificuldades

  • Encontrar erros em programas pode se tornar uma tarefa difícil e exigir muito tempo
  • Portanto, todo programador deveria procurar
    • Escrever código de boa qualidade
    • Estudar e aplicar técnicas que ajudam a evitar erros
      • Estar ciente das características das linguagens utilizadas
      • Escrever bons casos de testes
      • Usar boas ferramentas de depuração

Depuração

  • Depuração (debugging ou debug) é um procedimento para diagnóstico e correção de erros já detectados em um programa
  • Ela é muito útil pois permite ao programador
    • Monitorar a execução de um programa (passo a passo)
    • Ativar pontos de parada (linha, função, condição)
    • Monitorar os valores das variáveis
    • Alterar áreas de memória
    • Monitorar as chamadas de funções (stack trace)
    • Retroceder na execução do programa

Depuração

  • Em geral, mais da metade do tempo gasto no desenvolvimento de software é gasto com depuração
  • Apesar da boa utilidade, a depuração não é sempre a melhor alternativa
    • Certos tipos de programas não se dão muito bem com ela
      • Sistemas operacionais, sistemas distribuídos, múltiplos threads, sistemas de tempo real
    • Não é possível em algumas linguagens e ambientes
    • Pode variar muito de um ambiente para outro
    • Pode ser complicada para programadores iniciantes

  • Mas como vocês não são mais iniciantes!!!

Depuração

  • Uma solução alternativa à depuração se passa pelo uso criterioso dos comandos de impressão (printf, std::cout)
    • Estrutura de mensagens que mostra por onde o código está passando
    • Mas o código original é alterado para poder inserir estas mensagens (ruim)
  • Um grande interesse da depuração é que ela permite mostrar por onde o código está passando sem alterar o código original (bom)

Depurador GNU Debugger (GDB)

  • GDB é um depurador do GNU que suporta diversas
    linguagens de programação
    • C, C++, Objective-C, Java, Fortran, etc.
  • GDB foi criado por Richard Stallman em 1986
  • Ele é mantido pelo comitê GDB nomeado pela
    Free Software Foundation
  • Tal qual os compiladores GCC e G++ do GNU,
    o depurador GDB também pode ser integrado à
    diversos ambientes de desenvolvimento
    • NetBeans, Eclipse, Dev-C++, etc.

Passos para correção de erros

  • Quando um erro for detectado, refletir bem em quais pistas ele pode estar dando
    • Como ele pode ter acontecido?
    • Ele é parecido com algo que já foi visto?
    • O código foi modificado recentemente?
    • Há algo de especial na entrada que causou o erro?
  • Alguns poucos casos de teste e algumas poucas impressões podem em muitos casos ser suficientes para conseguir corrigí-lo

Passos para correção de erros

  • Mas se não há pistas, o melhor é pensar muito cuidadosamente sobre o que pode estar acontecendo
  • Tentar sistematicamente localizar o problema eliminando partes da entrada e do código
  • Explicar o código para outras pessoas
  • Usar um depurador para verificar a pilha de execução
  • Executar o programa passo a passo

Boas práticas para correção de erros

  • Usar as ferramentas de suporte que estão disponíveis
  • Procurar aprender com os próprios erros de modo a não repetí-los
  • Quando encontrar um erro, lembrar de eliminar possíveis erros parecidos localizados em outras partes do código
  • Tentar evitar que erros se repitam no futuro

Resumo da aula

  • Depuração é um procedimento para diagnóstico e correção de erros lógicos em programas que, dificilmente, são encontrados em tempo de desenvolvimento
  • Ela pode ser bastante eficaz desde que feita de forma sistemática e organizada
  • A melhor solução é, no entanto, escrever bom código que será menos propenso a erros, onde os erros existentes poderão ser de mais fácil detecção
