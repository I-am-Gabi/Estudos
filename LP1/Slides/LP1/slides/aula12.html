Biblioteca

  • Biblioteca é uma coleção de subprogramas (códigos e estruturas de dados) que provêm serviços para serem utilizados no desenvolvimento de software
  • Principal objetivo de uma biblioteca é disponibilizar código genérico e modular capaz de ser reutilizado por diversos programas

Compilação de programas em C++

  • A compilação de um programa em C++ envolve 3 passos principais:
    • Pré-processamento
      • Executa as diretivas #ifdef, #include, substitui macros, etc.
      • Gera arquivos temporários com código pré-processado
    • Compilação
      • Processa os arquivos temporários com código fonte pré-processado seguindo a gramática da linguagem
      • Gera arquivos objeto (.obj / .o) caso não ocorram erros
    • Linkedição
      • Faz a ligação de todos os arquivos de código-objeto gerados
      • Gera executável (.exe / .out) ou biblioteca dinâmica (.dll / .so)

Compilação de programas em C++

Pré-processador

  • O pré-processador é a primeira etapa da compilação, sendo responsável pelas seguintes tarefas:
    • Junção de linhas que foram separadas por escapes
    • Remoção de comentários substituido-os por espaços
    • Expansão de macros
    • Processamento das diretivas de pré-processamento
  • Os arquivos de cabeçalho (.h, .hpp, etc.) somente são compilados quando incluídos em algum arquivo de corpo (.c, .cpp, .cc, etc.)
  • A busca dos arquivos incluídos se faz da seguinte maneira:
    • #include <arquivo>: diretórios configurados no compilador
    • #include "arquivo": diretório do código fonte do programa

Compilador C++

  • O compilador C++ é responsável por verificar se o código fonte está bem construído segundo a gramática da linguagem
    • Erro de sintaxe (ex.: falta de um ;)
    • Erro de semântica (ex.: variável definida duas vezes)
  • Ele trata cada arquivo fonte como sendo uma unidade de compilação, compilando-a de maneira independente e ignorando a existência de qualquer outro arquivo
  • A única forma do compilador considerar um outro arquivo é através da diretiva #include, tratada pelo pré-processador

Compilador C++

  • O compilador precisa inserir no código objeto uma chamada para a função/método utilizada
    • Ele insere apenas uma anotação sobre a chamada, uma vez que a implementação pode ser definida em outro arquivo
  • Dessa forma, as chamadas de funções/métodos e acesso a variáveis globais em arquivos objetos são apenas temporárias

Linkeditor

  • O linkeditor (ou programa de ligações) é responsável por juntar todos os arquivos objetos e gerar o arquivo executável ou biblioteca dinâmica
  • A tarefa dele é substituir, nos arquivos objeto, todas as chamadas de funções/métodos e acessos a variáveis pelo endereço real de onde o item se encontra

Linkeditor

  • Também é tarefa do linkeditor juntar no executável qualquer biblioteca extra que for especificada
    • Um erro bem comum é usar alguma biblioteca externa e esquecer de linkeditar os seus arquivos .lib ou .a com o projeto
  • Um erro de linkedição que persegue muitos iniciantes é o seguinte:



    • Para o compilador não interessa se a função realmente existe, importando apenas se ela foi declarada
    • Mas o linkeditor não consegue encontrar a definição da função
void funcao();
int main() {
    funcao();  // Esta função não é encontrada
    return 0;
}

Tipos de bibliotecas

  • As bibliotecas podem ser classificadas pela maneira como são compartilhadas e ligadas:
    • Bibliotecas estáticas (união)
      • O código da biblioteca é incluído no programa
      • Temos uma cópia da biblioteca em memória para cada programa sendo executado
    • Bibliotecas dinâmicas (compartilhamento)
      • O código da biblioteca não é incluído no programa
      • Temos apenas uma única cópia da biblioteca em memória compartilhada por diversos programas sendo executados

Biblioteca estática

  • Bibliotecas estáticas são criadas com a união de arquivos objeto
  • O programador deve informar ao linkeditor sobre os arquivos da biblioteca a serem usados em um programa
  • Durante o processo de linkedição, o linkeditor busca a implementação das funções/métodos e variáveis globais utilizadas
    • Busca em todos arquivos objeto do programa
    • Busca em todos arquivos da biblioteca
  • Caso não encontre a implementação em lugar nenhum, teremos erro de linkedição

Biblioteca estática

  • Bibliotecas estáticas copiam o código das funções/métodos e suas dependências para o arquivo executável
  • O programa incorpora o conteúdo da biblioteca no executável, tornando os arquivos dela desnecessários após esta geração
    • Os arquivos da biblioteca não são requeridos na execução
    • Caso a biblioteca seja atualizada, será preciso gerar um novo executável
  • Tipos de arquivo:
    • .lib no Windows
    • .a no Linux

Biblioteca estática

  • Bibliotecas estáticas são carregadas na memória a cada vez que um programa é executado

Biblioteca dinâmica

  • As bibliotecas dinâmicas são quase como arquivos executáveis, mas não podem ser executadas diretamente
    • Precisam de que algum outro programa para carregar e invoquar suas funções/métodos
  • A geração de uma biblioteca dinâmica requer um linkeditor que faz quase o mesmo trabalho que na geração de um executável
  • Tipos de arquivo:
    • .dll (Dynamic Link Library) no Windows
    • .so (Shared Library) no Linux

Biblioteca dinâmica

  • Assim como em um executável comum, o linkeditor precisa obter informações sobre todas as bibliotecas requeridas

  • O linkeditor inclui no executável uma referência para a biblioteca dinâmica e todas suas funções utilizadas ficam apenas em stubs
    • Os stubs servem apenas para indicar ao sistema operacional que devem ser trocados por chamadas a bibliotecas dinâmicas
    • Isso permite que a versão de uma biblioteca possa ser trocada sem modificar o programa executável

Biblioteca dinâmica

  • Na execução de um programa, o sistema operacional se encarrega de checar quais bibliotecas dinâmicas precisam ser carregadas
  • Bibliotecas dinâmicas são carregadas apenas uma vez na memória






  • Após ter carregado as bibliotecas, o sistema operacional troca os stubs pelos endereços das funções reais
