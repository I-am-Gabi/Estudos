O que é um computador?

  • O computador é uma máquina de calcular extremamente eficaz
    • Consome informação (dispositivos de entrada)
    • Armazena informação (dispositivos de memória)
    • Transforma informação (dispositivos de processamento)
    • Produz informação (dispositivos de saída)
  • Informações são dados e programas usados para controlar o computador

Um pouco de História

  • ENIAC foi o primeiro computador digital de propósito geral
    • Universidade da Pensilvânia (1943-1945) por Mauchly e Eckert
    • Muito útil e viável economicamente
    • Grande dificuldade de reprogramação
    • Grande consumidor de espaço e de energia
    • Pouca disponibilidade de memória para armazenamento de dados
O ENIAC ocupava uma sala de 72 metros quadrados, pesava 30000 kg e usava mais de 18 mil tubos a vácuo, que eram muito pouco confiáveis e aqueciam demasiadamente, consumindo 160 kilowatts de potência. Apesar de suas 18 mil válvulas, ele podia armazenar apenas 20 números por vez. Mas graças à eliminação de engrenagens, era o mais rápido da época com uma velocidade do relógio interno de 100000 ciclos por segundo.

Um pouco de História

  • EDVAC foi o primeiro computador capaz de armazenar um programa num dispositivo de memória endereçável
    • Universidade da Pensilvânia (1946-1949) por Mauchly, Eckert e Von Neumann
    • Arquitetura revolucionária com grande repercussão acadêmica
      • Sistema de numeração binário
      • Dados e programa mapeados no mesmo
        espaço de endereços
      • Utilização de dispositivo de entrada/saída
    • Novo paradigma de concepção que ainda serve
      de base para os computadores atuais
O EDVAC ocupava uma sala de 45,5 metros quadrados, pesava 7850 kg e usava quase 6000 tubos de vácuo e 12000 diodos, consumindo 56 kilowatts de potência. O EDVAC recebeu várias atualizações, incluindo um dispositivo de entrada/saída de cartões perfurados em 1953, uma memória adicional em um cilindro magnético em 1954 e uma unidade de aritmética de ponto flutuante em 1958.

Eliminando ambiguidade e imprecisão

  • Para remover a imprecisão, é necessário:
    • Descrever o que deve ser feito de forma detalhada
      • Algoritmo
  • Para remover a ambiguidade, é necessário:
    • Usar uma especificação que defina regras de forma (sintaxe) e sentido (semântica)
      • Linguagem de programação

Algoritmo

  • Sequência precisa, sistemática e finita de passos ou instruções para realizar uma tarefa
    • Exemplo: Receita detalhada de preparação de um bolo
  • Pode ser escrito em linguagem natural
    • O computador também não entende
  • Deve seguir as seguintes propriedades
    • Garantia de término
    • Exatidão
    • Efetividade
- Garantia de término é quando o problema a ser resolvido possui condições específicas que, quando satisfeitas, a execução do algoritmo é encerrada e o problema é então tido como "resolvido". Além disso, estas condições devem ser satisfeitas após uma quantidade finita de tempo, a ser contada a partir do início da execução do algoritmo. - Exatidão é a intenção de que cada instrução no algoritmo deve ser suficientemente clara, de forma que não haja ambiguidade na interpretação da intenção. - Efetividade é quando cada instrução deve ser básica o suficiente para ser executada, pelo menos em princípio, por qualquer agente usando apenas lápis e papel.

Algoritmo

Cálculo do máximo divisor comum entre dois números inteiros x e y:

  1. Denomine x como o maior número e y como o menor número ;
  2. Divida x por y e denomine r como o resto ;
  3. Se r é igual a zero então o MDC é igual a y e a execução das instruções encerra aqui. Caso contrário, siga para a próxima instrução ;
  4. Atribua o valor de y a x e o valor de r a y ;
  5. Volte para o passo 2 .

Programa

  • Codificação de um algoritmo em uma linguagem de programação
  • Construção resultante da combinação de elementos simples
    • Regras estritas de sintaxe
      • Não pode escrever errado
    • Regras estritas de semântica
      • Não pode escrever besteiras
  • O computador também não entende, mas existem ferramentas para traduzir programas em linguagem de máquina
    • Compilador

Sintaxe

  • A sintaxe de uma linguagem diz quais são os elementos simples a serem utilizados e como eles podem ser combinados
    • Exemplo na língua portuguesa
      • Regra gramatical: sujeito, verbo e complemento
      • Erro de sintaxe: "Venha! Começando o jogo está"
    • Exemplo na linguagem de programação C
      • Regra gramatical: tipo nomeDaVariavel  =  valor ;
      • Erro de sintaxe: duracaoDoJogo  int  =  1 ;
  • Erros de sintaxe são facilmente detectáveis
    • O compilador nunca deixa passar um erro de sintaxe

Semântica

  • A semântica diz o que deve ser feito quando os elementos simples são encontrados, sejam sozinhos ou combinados
    • Exemplo na língua portuguesa
      • A veracidade de uma frase
      • Erros de semântica: "Eu sei trigonometria! π vale 7.1415"
    • Exemplo na linguagem de programação C
      • A veracidade de um código
      • Erros de semântica: π  =  7.1415 ;
  • Erros de semântica requerem uma análise de significado
    • O compilador não detecta
    • Pode nunca serem percebidos!
- Exemplos de erros de semântica: bug do ano 2000, várias catastrofes de aviões e foguetes - Comentar que a indústria sabe que após pronto e em uso um programa tem de 10 a 50 erros de semântica por cada 1000 linhas de código!
