Contexto

Modularidade

  • Estratégia para a construção de sistemas complexos a partir de pequenos subsistemas que podem ser desenvolvidos individualmente

Modularidade de programas

  • Grandes sistemas contêm geralmente muitas classes de objetos








  • Para facilitar a estruturação, leitura e manutenção do código, as classes devem ficar organizadas em diversos arquivos separados

Modularidade de programas

  • Por convenção da linguagem C++, a organização de classes de um programa orientado a objetos é feita da seguinte maneira:
    • Cada classe é construida fazendo uso de 2 arquivos, ambos sendo nomeados com o próprio nome da classe
    • Arquivo de cabeçalho com extensão .h (header)
      • Contém apenas a estrutura da classe (definição dos
        dados e das interfaces dos métodos membros)
    • Arquivo de corpo com extensão .cpp (C Plus Plus)
      • Contém a implementação dos métodos membros
        da classe

Modularidade de programas

  • Quando usamos uma biblioteca, não precisamos ler suas milhares de linha de código para saber o que ela é capaz de fazer



  • Ao invés disto, precisamos apenas ter contato com os cabeçalhos, que são arquivos bem pequenos, simples e explicativos
    • Eles descrevem como usar as funcionalidades da biblioteca sem se preocupar com a maneira na qual ela foi implementada

Exemplo de arquivo de cabeçalho (Ponto.h)

#ifndef _PONTO_H_  // Uso do pré-processador
#define _PONTO_H_  // Veremos já como isso funciona!!!

class Ponto
{
// Definição dos dados membros
private:
    int x, y;

// Definição dos métodos membros
public:
    Ponto();
    Ponto( int x, int y );

    int  getX     () const;
    int  getY     () const;
    int  distancia( const Ponto& ponto ) const;
    void imprimir () const;
};

#endif  // Fim da definição _PONTO_H_

Exemplo de arquivo de corpo (Ponto.cpp)

#include <iostream>
#include <cmath>
#include "Ponto.h"  // Inclusão do arquivo de cabeçalho da classe Ponto

Ponto::Ponto()
    : x( 0 ), y( 0 ) {}

Ponto::Ponto( int x, int y )
    : x( x ), y( y ) {}

int inline Ponto::getX() const {
    return x;
}

int inline Ponto::getY() const {
    return y;
}

int Ponto::distancia( const Ponto& ponto ) const {
    return std::sqrt( std::pow( x - ponto.x, 2 ) + std::pow( y - ponto.y, 2 ) );
}

void Ponto::imprimir() const {
    std::cout << x << ", " << y << std::endl;
}

Exemplo de programa principal (main.cpp)

#include <iostream>
#include "Ponto.h"  // Inclusão do arquivo de cabeçalho da classe Ponto

int main()
{
    Ponto ponto1;
    Ponto ponto2( 10, 5 );
    Ponto ponto3( 100, 50 );

    ponto1.imprimir();
    ponto2.imprimir();
    ponto3.imprimir();

    std::cout << "Distancia entre pontos 1 e 2: " << ponto1.distancia( ponto2 ) << std::endl;
    std::cout << "Distancia entre pontos 1 e 3: " << ponto1.distancia( ponto3 ) << std::endl;
    std::cout << "Distancia entre pontos 2 e 3: " << ponto2.distancia( ponto3 ) << std::endl;

    return 0;
}

Arquivos de cabeçalho em C++

  • Podem conter declarações de classes, estruturas, tipos, variáveis globais, protótipos de funções e macros
  • Não podem conter o método principal main de um programa

  • Eles são incluídos em um programa através da diretiva de pré-processamento #include seguida do nome do arquivo
    • A inclusão copia o conteúdo de um arquivo em outro
  • A fim de evitar duplicidade de cópias do código a cada nova inclusão, os cabeçalhos precisam ser identificados
    • A identificação é feita através da definição de um nome via o comando #define do pré-processador

Pré-processador em C++

  • O pré-processador em C++ faz com que um determinado texto seja processado e inserido no código
  • Todos os comandos do pré-processador iniciam por um #
  • O comando #include copia o conteúdo de um arquivo para outro através de duas formas:
    • Quando usando arquivos das bibliotecas
      • #include <stdio.h> (biblioteca padrão de C)
      • #include <iostream> (biblioteca padrão de C++)
    • Quando usando arquivos do próprio usuário
      • #include "Ponto.h" (classe definida pelo usuário)

Pré-processador em C++

  • O comando #define substitui palavras por valores, atuando como uma constante
    • #define PI 3.14159 permite substituir PI pelo valor 3.14159
  • O comandos #ifdef, #ifndef, #endif permitem evitar que partes do código sejam inseridas no programa
    • Úteis nos arquivos de cabeçalho para evitar duplicidade
    • #include não verifica se um arquivo já foi incluido no programa
    • #ifndef (If Not Defined) permite-nos fazer essa verificação
#ifndef _NOME_DO_ARQUIVO_H_  // Evita a redefinição dos membros do arquivo
#define _NOME_DO_ARQUIVO_H_  // Início da definição de _NOME_DO_ARQUIVO_H_
                             // Código do arquivo deve se localizar aqui
# endif                      // Fim da definição de _NOME_DO_ARQUIVO_H_

Arquivos de corpo e do método principal

  • Arquivos com extensão .cpp que implementam os métodos definidos nas classes presentes nos arquivos de cabeçalho

  • Necessitam a inclusão dos arquivos de cabeçalho
    • #include <iostream> (biblioteca padrão de C++)
    • #include "Ponto.h" (classe definida pelo usuário)

  • Esta inclusão somente é efetuada uma vez no programa devido a verificação do identificador feita no próprio arquivo de cabeçalho

Projetos em C++

  • Um programa que usa diversos arquivos deve compilar todos eles a fim de gerar o código executável
  • Para evitar a necessidade de compilar manualmente os arquivos, podemos criar um arquivo de projeto para associá-los

  • Arquivos de projeto podem ser criados da seguinte maneira:
    • Manualmente através de um arquivo de makefile que contém as instruções de como construir o executável
    • Automaticamente com ambientes de desenvolvimento integrado que associam arquivos para compilá-los em conjunto
  • Obs: Ver tutoriais no SIGAA para criação de projetos

Namespace

  • Mecanismo para expressar agrupamentos lógicos de modo a evitar confusões quando trabalhamos com diversas bibliotecas
  • Bastante útil quando queremos criar classes ou funções com nomes já existentes em outras partes do código, sem que estas entrem em conflito com o código existente
  • Sintaxe:
    • namespace nomeDoNamespace { corpoDoNamespace }
  • Exemplo:
    • Agrupamento de um conjunto de classes
      que realizam operações geométricas
namespace geometria
{
   class Ponto { ... };
   class Circulo { ... };
   class Poligono { ... };
}

Acesso ao namespace

  • Desde o início do semestre nós usamos namespaces
    • Exemplo: namespace std (standard) da biblioteca padrão
  • O acesso aos membros de um namespace pode ser feito assim:
    • Usando o operador ::
    • Usando o commando using namespace
#include <iostream>
#include <algorithm>

int main()
{
    int x, y;
    std::cin >> x >> y;
    std::cout << std::max( x, y ) << std::endl;
    std::cout << std::min( x, y ) << std::endl;
    return 0;
}

#include <iostream>
#include <algorithm>
using namespace std;

int main()
{
    int x, y;
    cin >> x >> y;
    cout << max( x, y ) << endl;
    cout << min( x, y ) << endl;
    return 0;
}

Exemplo de uso de namespaces

namespace mat1
{
    template < class T >
    T max( T a, T b )
    {
        return ( a > b ) ? a : b;
    }
}

namespace mat2
{
    template < class T >
    T max( T a, T b )
    {
        return ( a > b ) ? a : b;
    }
}

void calcular()
{
    using namespace std;
    max( 10, 5 );
}

#include <algorithm>

int main()
{
    mat1::max( 10, 5 );
    calcular();
    mat2::max( 10, 5 );
    std::max( 10, 5 );

    return 0;
}

Problema com acesso ao namespace

  • No exemplo anterior, os namespaces mat1, mat2 e std contém a definição da mesma função max
  • Se não usarmos esses namespaces com cuidado, poderemos gerar um conflito no código do programa
#include <iostream>
#include <algorithm>
using namespace std;
using namespace mat1;
using namespace mat2;

int main() {
    int x, y;
    cin >> x >> y;

    // Quais funções serão chamadas? As do namespaces mat1, mat2 ou std?
    cout << max( x, y ) << endl;  // erro de conflito

    return 0;
}

Problema com acesso ao namespace

  • Para resolver este tipo de conflito, podemos ser mais específicos indicando que apenas alguns membros do namespace sejam visíveis
#include <iostream>
using std::cout;
using mat1::max;

int main()
{
    int x, y;
    std::cin >> x >> y;

    // A função max do namespace mat1 será chamada
    cout << max( x, y ) << std::endl;

    return 0;
}

Vantagens da modularidade

  • Código modular permite ser desenvolvido e testado uma só vez, embora possa ser usado em várias partes de um programa
  • Permite a criação de bibliotecas que pode ser usada em diversos programas e programadores
  • Permite economizar memória uma vez que o módulo utilizado é armazenado uma única vez, mesmo que utilizado em diferentes partes do programa
  • Permite ocultar código uma vez que apenas a estrutura das classes ficam disponíveis para outros programadores
